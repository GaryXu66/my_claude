## 一、网关授权领域

### api-gateway

**业务定位**：API统一网关，承担所有外部请求的接入、路由、鉴权、限流等职责。

**核心业务逻辑**：
- **多粒度鉴权**：支持应用级(app)、团队级(team)、资源组密钥级(resGroupSecret)等粒度的权限控制
- **多认证方式**：OAuth2.0、JWT、Session、Ticket等多种认证方式并存
- **动态路由**：基于Netflix Zuul实现动态路由，支持Motan RPC泛化调用
- **流量管控**：请求限流、熔断降级、灰度发布
- **会话管理**：基于Redis的分布式会话，支持多端登录控制
- **20个核心Service**：OAuth2Service、JWTAuthService、GatewayAdminService、TicketService、WxMiniService等

**核心数据库表**：
| 表名 | 业务含义 |
|------|---------|
| T_GatewayService | 网关服务配置，记录服务基本信息 |
| T_GatewayEndpoint | 网关端点配置，记录API接口（innerPath、outerPath、authTypes、reqPerSec限流） |
| T_App | 应用信息，存储appId、secret、RSA密钥 |
| T_Authority | 授权信息，存储appId与资源的授权关系、accessToken、refreshToken |
| T_Resource | API资源，记录URI、方法、认证类型 |
| T_ResourceGroup | 资源分组，用于资源的逻辑分组管理 |
| T_GroupSecret | 分组加密密钥，用于资源分组的安全访问 |

**关键技术特性**：
- 使用Guava Cache缓存access_token（50K max, 5min TTL）
- 基于RSA加密保护敏感信息
- MongoDB存储网关配置
- Redis Sentinel保证高可用
- Zookeeper实现配置管理和微服务注册发现

### authserver

**业务定位**：OAuth2授权服务器，负责应用注册、Token生成和验证、权限控制等核心授权服务。

**核心业务逻辑**：
- **应用注册**：创建和管理应用信息（AppService）
- **Token管理**：生成access_token、refresh_token，验证和刷新token（AuthorityService）
- **权限控制**：管理应用与资源的授权关系（AppAuthorityService）
- **资源管理**：管理API资源的URI和方法（ResourceService）
- **纯RPC服务**：不提供Web接口，仅暴露Motan RPC服务

**核心数据库表**：
| 表名 | 业务含义 |
|------|---------|
| T_App | 应用信息（appId、secret、RSA密钥、eid） |
| T_Authority | 授权信息（accessToken、refreshToken、expireTime默认30天） |
| T_Resource | API资源（uri、method、authType） |
| T_ResourceGroup | 资源分组 |
| T_GroupSecret | 分组密钥 |
| T_AccessLog | 访问日志 |

## 十七、序列号领域

### sequence-service

**业务定位**：分布式序列号生成服务，基于Snowflake算法实现全局唯一ID生成。

**核心业务逻辑**：
- **单个序列号获取**：`/sequence?key=your-named-key`
- **批量序列号获取**：`/sequence/list?key=your-named-key&count=20`
- **按业务Key隔离**：支持不同业务的序列号生成
- **高并发、高可用**：支持Docker部署，多实例负载均衡

**技术栈**：Spring Boot + Snowflake算法
