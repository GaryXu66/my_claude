# 任务管理

## 快速开始

**核心原则**：必须按模板格式生成，支持中断恢复，实时状态更新

**文档位置**：`docs/task/{YYYYMMDD}_{中文模块名}_任务.md`

**模板位置**: 必须严格按照 [任务文档模板](templates/task-doc.md) 模板格式生成

**状态标记**：📋 待办 / 🚧 进行中 / ✅ 已完成 / 🚫 阻塞 / ❌ 取消

详细规范请参考模板，[任务文档模板](templates/task-doc.md)

---

## 操作工作流

### 1️⃣ 初始化任务文档

**触发**：用户首次输入需求

-**操作**：创建文档骨架，只填写这些章节： `流程状态总览`、`详细任务清单`的初始任务、`下一步指令`、`任务日志`

### 2️⃣ 创建任务清单

**触发**：方案确认后

**操作**：补充`详细任务清单`。

### 3️⃣ 更新任务状态

**触发**：任务进展时

**操作**：更新状态标记，记录`下一步指令`、`任务日志`

### 4️⃣ 查看任务进度

**触发**：用户询问进度

**操作**：汇报整体进度、下一步指令和待完成任务

### 5️⃣ 任务归档

**触发**：功能开发完成

**操作**：更新状态，记录到`任务日志`，自行确定或询问用户下一步指令

## 质量检查

### 文档创建后检查
- [ ] 是否按照模板格式生成
- [ ] 是否包含流程状态总览与产物清单
- [ ] 是否包含`下一步指令`
- [ ] 任务状态是否可跟踪（使用状态标记）

### 使用过程检查
- [ ] 状态更新是否及时
- [ ] 是否需要增加新的任务项
- [ ] 是否需要变更 `下一步指令`
- [ ] 任务重大变更是否及时记录到 `任务日志`
