# 任务文档模板

## 使用规范

### 核心原则
- **必须按模板格式生成**：不能仅凭内容完整性判断，必须符合可跟踪格式
- **支持中断恢复**：任务状态清晰，对话中断后能快速恢复进度
- **实时状态更新**：任务进展时及时更新状态标记
- **不写细节**：仅记录流程状态、产物。尽可能用简短的文字+链接减少视觉文字

### 状态标记说明
- 📋 待办任务
- 🚧 进行中任务
- ✅ 已完成任务
- 🚫 阻塞任务
- ❌ 取消任务

### 章节说明
- **流程状态总览**：模块整体核心任务的总览和主要文档、代码的链接
- **详细任务清单**：详细的任务清单，重点知道还有哪些事情没有做，哪些任务阻塞了
- **下一步指令**：重点描述当前首要的任务，此任务的上下文，辅助中断后知道如何继续
- **任务日志**：任务完成、阻塞、重启和取消 记录到这里，方便后续追查进展

### 更新规范

**更新时机**：
- **需求建立**：需求基本确定后，应创建流程状态总览，和部分详细任务
- **设计建立**：在设计基本确定后，应补充更加详细的详细任务
- **需求和设计的变更**：需求和设计变更时，应重新回顾流程状态总览，任务清单检查是否需要增改
- **开始任务**：📋 → 🚧
- **完成任务**：🚧 → ✅
- **遇到阻塞**：🚧 → 🚫 并说明原因
- **任务取消**：📋 → ❌
- **阶段推进**：同步更新流程状态、产物清单、任务清单、下一步指令
